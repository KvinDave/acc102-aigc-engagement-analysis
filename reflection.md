### 1. Project aim and rationale
This project investigated a practical platform question: which factors are associated with stronger engagement for AIGC-related posts on Xiaohongshu, also known as Red Note? I chose this topic because it sits at the intersection of social-media analytics, content strategy, and data storytelling. Rather than producing a traditional essay, I wanted to build a usable data product that could help a creator or marketer decide what type of post is most likely to perform well. That meant the project needed to do more than display charts. It had to take raw platform data, turn it into interpretable evidence, and then present that evidence in a form that a non-technical user could explore.

### 2. What I built and why
I built three connected outputs: a reproducible Jupyter notebook, a structured GitHub repository with documentation, and an interactive Streamlit dashboard. The notebook is the analytical backbone of the project. It documents data loading, cleaning, feature engineering, visualisation, and statistical testing in a logical top-to-bottom sequence. The GitHub repository acts as the public-facing project package, allowing a marker or external viewer to understand the problem, reproduce the workflow, and locate the app quickly. The Streamlit dashboard translates the notebook into a more user-centred product by letting the audience filter by category, post type, year range, and tag count, then inspect the resulting engagement benchmarks.

This distinction-upgrade version improved the project in one important way: it moved from purely descriptive analysis to evidence-supported interpretation. In the earlier version, I had already identified strong descriptive patterns, but I realised that stronger submissions should also show how reliable those patterns appear to be. To address that, I added non-parametric statistical tests and a lightweight benchmark model using log-transformed engagement. This did not turn the assignment into a full predictive modelling project, but it did strengthen the analytical credibility of the conclusions.

### 3. Key technical decisions
The first technical challenge was encoding. The posts dataset uses GBK rather than UTF-8 because it contains Chinese text. Failing to identify that would make the notebook break immediately, so handling it correctly was essential for reproducibility. The second challenge was count formatting. Several engagement metrics appear in Chinese shorthand, such as `2.2万`, which needed to be parsed into numeric values before any analysis could take place. I wrote a parser to convert these values safely and then constructed a composite engagement measure from likes, saves, comments, and shares.

A further methodological choice was to use the median as the main summary statistic. Social-media engagement data is highly skewed because a small number of viral posts can distort the mean. The median gave a better picture of typical post performance. I also log-transformed engagement in the benchmark model to reduce the effect of extreme outliers and to make coefficient comparisons more stable.

### 4. What I learned from the analysis
The strongest insight was that category matters substantially. Print-related AIGC posts achieved the highest median engagement, far ahead of categories such as automotive content. That suggests the platform’s audience composition strongly shapes which AIGC use cases succeed. I also found that video posts clearly outperform normal posts, which is consistent with wider short-form content trends but still important to demonstrate with platform-specific evidence. Another useful finding was the inverted-U relationship between tag count and engagement: too few tags limit discoverability, while too many appear to weaken performance. In the filtered dashboard, the best-performing range consistently centred on roughly 9+ tags.

Adding statistical tests improved my confidence in the interpretation. The Mann–Whitney U test for video versus normal posts returned a very small p-value (6.821e-14), and the Kruskal–Wallis test across categories did the same (1.41e-90). These tests do not prove causation, but they do suggest that the observed differences are unlikely to be random noise within this sample.

### 5. Limitations and next improvements
The project still has meaningful limitations. Most importantly, the dataset does not contain follower counts or other creator-level controls. That means some of the observed engagement differences could reflect creator popularity rather than post characteristics alone. The dataset is also observational and platform-specific, so the results should be understood as association rather than causal proof. In a future iteration, I would like to merge the posts file with the comment files, incorporate sentiment analysis, and test whether save-heavy posts differ systematically from like-heavy posts. I would also add a small prediction module in the app so that users could input their own planned post characteristics and receive a benchmark estimate.
